
# Trabalho 3

**A descrição está completa (acho). Dúvidas ou omissões serão esclarecidas aqui. Tem um acréscimo previsto para depois de vermos alocação dinâmica de memória.**

Implemente um programa de gerenciamento de notas, de acordo com as instruções abaixo.

   A implementação deve ser colocada em **um** arquivo, chamado `l1-t3-fulano.c` (com "fulano" substituído pelo que está na planilha de informações dos alunos).
   **Anexe esse arquivo** (não cole o conteúdo no texto da mensagem, não anexe uma imagem com o arquivo) em um e-mail e envie para `benhur+l126a@inf.ufsm.br`, com assunto `entrega de l1-t3-fulano`, até o prazo limite.

Uma nota é um pequeno texto (limitado a um número máximo de bytes, que não pode ser inferior a 100), além de informações adicionais que permitem sua organização.
O programa deve gerenciar um número qualquer de notas, limitado a um máximo (que não pode ser inferior a 100).
Cada nota deve ser implementada em um registro.
O conjunto de notas deve ser mantido em um vetor desses registros.
A ordem das notas no vetor é uma informação importante, e deve ser mantida.
No início da execução, o vetor deve ser inicializado com o conteúdo de um arquivo.
O programa deve atualizar esse arquivo, para que as notas não sejam perdidas quando o programa encerrar sua execução.

Durante a execução, o programa permite ao usuário
- acessar o conteúdo das notas;
- buscar alguma nota;
- alterar a ordem de alguma nota no vetor;
- alterar o texto de uma nota;
- alterar alguma informação adicional de uma nota;
- remover uma nota;
- criar uma nova nota.

Informações adicionais de uma nota:
- cor;
- retângulo com tamanho e posição na tela;
- etiqueta - 3 letras que servem para categorizar a nota.

A cor pode ser um registro com os valores dos componentes rgb entre 0 e 255.
O retângulo deve ser um registro, contendo um ponto (canto superior esquerdo) e um tamanho (largura e altura).

Cada aluno pode escolher se suportará todas as informações adicionais ou um subconjunto delas, que pode ser:
- somente etiquetas;
- somente cor + retângulo.
A primeira opção é restrita aos alunos com deficiência visual que dificulte muito a manipulação/visualização dos retângulos.

## Formato do arquivo

O formato do arquivo de notas deve ser se acordo com o que segue. Caso o aluno opte por não suportar alguma das informações adicionais, ainda assim deve seguir o mesmo formado do arquivo, e essas informações devem ser mantidas no vetor e gravadas no arquivo. O arquivo contém uma nota por linha, com os seguintes campos, separados por um ou mais espaços:
- 3 caracteres com a etiqueta;
- cor rgb: 3 números entre 0 e 255, codificados em decimal, separados por um ou mais espaços;
- retângulo: 4 numeros inteiros codificados em decimal, representando as cordenadas x e y e a largura e altura;
- o texto da nota, entre aspas duplas (**"**);
- um número qualquer de caracteres, que devem ser ignorados;
- o caractere de fim de linha ('\n');

Os caracteres válidos para uma etiqueta são as letras maiúsculas e os dígitos, ASCII.

Os caracteres válidos para o texto de uma nota são os caracteres imprimíveis da tabela ASCII (códigos entre 32 e 126) exceto aspas duplas (").

Opcionalmente, podem ser suportados caracteres unicode codificados em UTF8 no texto das notas. A descrição abaixo menciona o que deve ser considerado caso se suporte UTF8 ou não.

Se o texto de alguma nota contiver mais caracteres que o suportado pelo programa, os caracteres a mais devem ser ignorados, mas o programa deve funcionar corretamente. A linha deve ser copiada para o final de um arquivo de linhas com problemas.
Caso não suporte UTF8, a leitura deve ignorar caracteres com código superior a 126 que existam no texto, entre aspas.

Se uma linha não estiver corretamente formatada, deve ser copiada para o final do arquivo de linhas com problemas, e ignorada pelo programa (que deve continuar lendo as demais linhas).

Exemplo do conteúdo do arquivo
```
LRC 200 50 30 20 3 15 10 "Lavar a roupa"
A12 180 50 50 1 10 10 3 "limpar gambiarra da leitura"
CCJ 100 50 205 50 1 20 20 "arroz oleo sal sardinha"
A12 255 10 10 1 1 5 1 "T3!"@#!"!
A12 180 50 50 1 7 10 2 "refazer laco principal"
```

## Organização do programa

O programa deve ter um estrutura modal, controlado pelo teclado.
Um programa modal tem vários modos de funcionamento possíveis (por exemplo, modo de edição e modo de navegação), e a cada momento está em um desses modos.
Em um modo, o usuário pode realizar um conjunto de ações, selecionáveis pelo teclado; entre elas sempre está a ação de trocar de modo.
Cada modo é implementado como um laço:
```c
   enquanto estiver no modo tal:
   - desenha a tela
   - lê o teclado
   - executa a ação correspondente
```

Deve ter uma função para cada modo do operação do programa, implementando um laço como esse.
Deve ter uma função de desenho de tela para cada modo, que é chamada no laço correspondente.
Todo comando de saída para a tela deve estar nas funções de desenho de tela.
Deve ter uma função para cada ação possível.

O programa tem um laço principal assim:
```
  enquanto o modo não for "terminar"
  - chama a função correspondente ao modo atual
```

O vetor de notas e demais informações úteis para o funcionamento do programa devem ser mantidos em um registro, que deve ser declarado na `main` e passado por referência para as funções que necessitem.
Na inicialização do programa, esse registro é inicializado. Nessa inicialização, o vetor de notas é lido do arquivo para esse registro.

## Modos de funcionamento do programa

O programa pode estar em um entre vários modos do operação, descritos abaixo.
Na inicialização do programa, deve ser selecionado o modo principal.
Os modos são:
- principal, onde ocorre a navegação entre as notas;
- edição de texto, onde se pode alterar o texto de uma nota;
- edição de etiqueta, onde se pode alterar a etiqueta de uma nota (se suportado);
- edição de cor, onde se pode alterar a cor de uma nota (se suportado);
- edição do texto de busca, onde se pode alterar o texto de busca;
- edição da etiqueta de busca, onde se pode alterar a etiqueta de busca (se suportado);

### Modo principal

Esse modo permite a navegação entre as notas.

Se existir um "texto de busca", as notas cujo texto não contiver o texto de busca devem ser ignoradas.
Se existir uma "etiqueta de busca", as notas cuja etiqueta não iniciar com o valor da etiqueta de busca devem ser ignoradas.

Uma das notas pode ser a "nota corrente". Sua escolha depende do suporte ou não a retângulos ou etiquetas, veja a seção correspondente abaixo.

#### Tela

Depende do suporte a cores, retângulos, etiquetas, veja na seção correspondente.

#### Ações

As ações nesse modo comuns a todas as implementações estão abaixo. Veja também as ações associadas ao suporte a cores, retângulos, etiquetas nas seções seguintes.

- **i** — muda a nota corrente para o início do vetor, mantendo a ordem relativa das demais notas;
- **f** — muda a nota corrente para o final do vetor, mantendo a ordem relativa das demais notas;
- **del** — remove a nota corrente, se houver; ela passa a ser a "última nota removida";
- **ins** — reinsere a "última nota removida", se houver; ela deve ser colocada no final do vetor. Após a inserção, passa a não existir "última nota removida".
- **n** — cria uma nova nota, no final do vetor, com conteúdo padrão;
- **g** — grava o arquivo de notas;
- **e** — muda o modo de operação para edição do texto da nota;
- **b** — muda o modo de operação para edição do texto de busca;

#### Modo principal com suporte a retângulos e cores

Uma posição na tela corresponde ao "cursor".
A nota nessa posição é considerada a "nota corrente".
Caso haja mais de uma nota na posição do cursor, a nota corrente é a que estiver mais para o final do vetor de notas, entre as que estão nessa posição.
Se não existir nota na posição do cursor, não existe nota corrente.

Nas ações de inserção de nota (nova ou reinserção de nota removida), ela deve ser posicionada de forma a conter a posição do cursor.

##### Tela

A tela deve mostrar as notas, cada uma em seu retângulo, com o fundo na cor da nota, o texto dentro do retângulo, e a etiqueta, se suportada, em um dos cantos do retângulo. Se a cor de fundo for escura (média do rgb inferior a 120), o texto deve ser em uma cor clara e vice versa.
Caso o texto não caiba no retângulo, o que não couber não aparece.
O espaço do retângulo deve poder ser totalmente utilizado pelo texto, não pode usar só uma linha. Veja abaixo (os +-| são só para delimitar, não é necessário imprimir assim).
```
       OK                         NOK
    +-------+                  +-------+
    |texto q|                  +       +
    |ue nao |          texto que nao cabe na nota
    |cabe na|                  |       |
    +-------+                  +-------+ 
```

As notas devem ser desenhadas na ordem em que estão no vetor, de forma que, se houver sobreposição, as notas mais próximas ao final do vetor pareçam sobre as anteriores.

O cursor deve ser posicionado na posição do cursor após o desenho do restante da tela.

A nota corrente deve ser reconhecível facilmente.

Caso suporte UTF8, atenção para a diferença entre caracteres e bytes, para ver o que cabe em cada linha.

##### Ações

As ações nesse modo, além das gerais, são:
- **setas** — move o cursor uma posição na direção da seta, se não estiver na borda;
- **shift+setas** — move o retângulo da nota corrente (se existir e se houver espaço para o movimento); move o cursor uma posição, se não estiver na borda;
- **ctrl+setas** — aumenta o retângulo da nota corrente na direção da seta, sem mover o outro lado (se existir nota sob o cursor e se houver espaço para o aumento);
- **alt+setas** — diminui o retângulo da nota corrente na direção da seta, sem mover o outro lado (se existir nota sob o cursor e se o tamanho for maior que o mínimo); se o retângulo sair do cursor, move o cursor uma posição;
- **p** — posiciona o cursor de modo que fique sobre a nota mais próxima do final do vetor, se houver;
- **c** — muda o modo de operação para edição de cor;

#### Modo principal com suporte a etiquetas

Se implementar também suporte a retângulos e cores e houver contradição com o que está aqui, tem preferência a descrição com suporte a retângulos.

No início da execução do programa, e cada vez que executou no modo de edição de texto de busca ou de etiqueta de busca, a nota corrente passa a ser a que está mais próxima do final do vetor, que não seja ignorada pelo texto ou etiqueta de busca.

##### Tela

Deve no mínimo mostrar a nota corrente e o número de notas não filtradas pelo texto ou etiqueta de busca.
Caso mostre também outras notas, deve ficar claro qual é a corrente.
As notas filtradas não devem ser mostradas.

##### Ações

- **seta para esquerda** - muda a nota corrente para a nota não filtrada que tem o maior índice no vetor menor que o índice da nota corrente atual (seleciona a próxima nota não filtrada); se não hover, mantém a nota corrente;
- **seta para direita** - muda a nota corrente para a nota não filtrada que tem o menor índice no vetor maior que o índice da nota corrente atual (seleciona a próxima nota não filtrada); se não houver, mantém a nota corrente;
- **Home** — muda a nota corrente para a nota não filtrada de menor índice;
- **End** — muda a nota corrente para a nota não filtrada de maior índice;
- **t** — muda o modo de operação para edição de etiqueta;
- **B maiúsculo** — muda o modo de operação para edição da etiqueta de busca;

### Modo de edição do texto

Esse modo permite a alteração do texto da nota corrente.
Ao entrar neste modo, o texto da nota deve ser copiado para um texto de edição, que é o que será editado neste modo.
Se a edição for confirmada, o texto da nota é substituído pelo texto editado.
Se a edição for cancelada, o texto original não é alterado.

Caso não exista nota corrente, altera o modo para principal e retorna.

#### Tela

A tela contém no mínimo o texto sendo editado, com o cursor no ponto de edição dentro desse texto.

#### Ações

Abaixo estão as ações nesse modo, para cada tecla tratada:
- **Enter** — copia o texto sendo editado para o texto da nota atual, muda para o modo principal
- **Esc** — muda para o modo principal
- **Backspace** — se a posição do cursor for maior que 0, subtrai um dessa posição e remove o caractere na nova posição
- **Del** — se o cursor estiver em uma posição inferior ao tamanho do texto, remove o caractere na posição do cursor
- **Seta esquerda** — se a posição do cursor for maior que 0, subtrai um dessa posição
- **Seta direita** — se a posição do cursor for menor que o tamanho do texto, soma 1 nessa posição

- **Caractere válido para o texto** — se houver espaço, insere o caractere na posição do cursor
- **Outra tecla** — ignora

#### Opcional

##### Submodo de sobrescrever o texto

No desenho, usa um cursor diferente se estiver inserindo ou sobrescrevendo.
Na entrada de caractere válido, insere ou sobrescreve de acordo com o submodo

- **Ins** — altera o submodo entre inserir e sobrescrever
- **Home** — vai para a posição 0
- **End** — vai para a posição correspondente ao tamanho do texto

##### Suporte a caracteres UTF8

Um caractere acentuado é codificado em 2 bytes em UTF8 (outros caracteres especiais podem usar até 4 bytes).
O primeiro byte de um caractere não ASCII tem código entre 128+64 e 255, os demais bytes têm código entre 128 e 128+63.
Forma simples de implementar suporte:
- Reserve espaço no vetor para até 3 bytes a mais, decida se permite a inclusão ou não depois de incluir.
- A posição do cursor é mantida por 2 números, a posição em caracteres (que serve para a tela) e a posição em bytes (que serve para acesso ao vetor).
- Quando o cursor avança, incrementa os dois, mas enquanto estiver sobre um byte de continuação, continua incrementando o contador de bytes.
- Quando o cursor recua, decrementa os dois, mas enquanto estiver sobre um byte de continuação, continua decrementando o contador de bytes.
- Quando remover um caractere, remova também os bytes de continuação. Use remoção e depois inserção para implementar substituição.
- Para implementar "End", avança de um em um.

Os caracteres não ASCII são lidos no formato UTF8, um byte por vez. Se não suportar esses caracteres, ignore os caracteres lidos com código entre 128 e 255. Se suportar, atenção na leitura, bytes com código entre 128 e 128+63 são bytes de continuação, devem ser armazenados mas não ocupam espaço na tela quando impressos.

### Modo de edição da etiqueta

Existe um valor de etiqueta de edição.
Esse modo permite a alteração desse valor, e sua cópia para notas.

Ao entrar neste modo, caso haja uma nota corrente, o valor da etiqueta da nota corrente é copiado para o valor da etiqueta de edição.

#### Tela

A tela contém no mínimo a etiqueta sendo editada, com o cursor no final.

#### Ações

Abaixo estão as ações nesse modo, para cada tecla tratada:
- **Esc** — muda para o modo principal;
- **Backspace** — remove o último caractere da etiqueta sendo editada, se houver;
- **Caractere válido para a etiqueta** — acrescenta ao final da etiqueta, se couber;
- **Enter** — se a etiqueta sendo editada tiver 3 caracteres, altera a etiqueta da nota corrente e muda para o modo principal;
- **Shift+Enter** — se a etiqueta sendo editada tiver 3 caracteres, altera a etiqueta de todas as notas visíveis e retorna ao modo principal;
- **Outra tecla** — ignora;


### Modo de edição de cor

Existe uma cor sendo editada.
Este modo permite alterar essa cor, e colocá-la em notas.

Ao entrar neste modo, se houver uma nota corrente, a cor sendo editada é alterada para a cor da nota corrente.

#### Tela

Mostra a cor sendo editada, e o valor de seus componentes. Um desses componentes é o selecionado e isso deve ser visível.

#### Ações

- **e** ou **r** — seleciona componente vermelho;
- **v** ou **g** — seleciona componente verde;
- **a** ou **b** — seleciona componente azul;
- **seta para cima** — aumenta o valor do componente selecionado, se for menor que 255;
- **shift seta para cima** — aumenta bastante o valor do componente selecionado;
- **seta para baixo** — diminui o valor do componente selecionado selecionado, se for maior que 0;
- **shift seta para baixo** — diminui bastante o valor do componente selecionado;
- **dígito** — se a última tecla foi um dígito, acrescenta e dígito digitado ao final do valor do componente selecionado, se não passar de 255; senão, substitui o valor do componente selecionado pelo valor o dígito;
- **seta para direita** — seleciona o próximo componente;
- **seta para esquerda** — seleciona o componente anterior;
- **enter** — coloca a cor editada na nota corrente; muda para modo principal;
- **shift+enter** — coloca a cor editada em todas as notas visíveis; muda para o modo principal;
- **esc** — muda para modo principal;


### Modo de edição do texto de busca

Existe um texto de busca.
Este modo permite alterar esse texto.

#### Tela

Mostra no mínimo o texto de busca, com o cursor na posição do cursor.

Ver comentário sobre suporte a UTF8 no modo de edição de texto da nota.

#### Ações

- **esc** — esvazia o texto de busca e muda para o modo principal;
- **enter** — muda para o modo principal;
- **caractere válido para texto** — acrescenta ao texto de busca, se couber;
- **backspace** — se o cursor não estiver no início, recua o cursor e remove o caractere do cursor;
- **del** — se o cursor não estiver no final, remove o caractere do cursor;
- **setas dir/esq** — move o cursor, se puder;
- **home** — cursor = 0;
- **end** — altera o cursor para o tamanho do texto de busca;

### Modo de edição da etiqueta de busca

Existe um texto de busca.
Este modo permite alterar esse texto.

#### Tela

Mostra no mínimo a etiqueta de busca, com o cursor no final.

#### Ações

- **esc** — esvazia a etiqueta de busca e muda para o modo principal
- **enter** — muda para o modo principal
- **caractere válido para etiqueta** — acrescenta ao final da etiqueta de busca, se couber
- **backspace** — remove o caractere final da etiqueta de busca, se houver
