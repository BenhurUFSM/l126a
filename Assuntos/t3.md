Implemente um programa de gerenciamento de notas, de acordo com as instruções abaixo.

A implementação deve ser colocada em **um** arquivo, chamado `l1-t3-fulano.c` (com "fulano" substituído pelo que está na planilha de informações dos alunos).
**Anexe esse arquivo** (não cole o conteúdo no texto da mensagem, não anexe uma imagem com o arquivo) em um e-mail e envie para `benhur+l126a@inf.ufsm.br`, com assunto `entrega de l1-t3-fulano`, até o prazo limite.

Uma nota é um pequeno texto (limitado a um número máximo de bytes, que não pode ser inferior a 100), além de informações adicionais que permitem sua organização.
O programa deve gerenciar um número qualquer de notas, limitado a um máximo (que não pode ser inferior a 100).
Cada nota deve ser implementada em um registro.
O conjunto de notas deve ser mantido em um vetor desses registros.
A ordem das notas no vetor é uma informação importante, e deve ser mantida.
No início da execução, o vetor deve ser inicializado com o conteúdo de um arquivo.
O programa deve atualizar esse arquivo, para que as notas não sejam perdidas quando o programa encerrar sua execução.

Durante a execução, o programa permite ao usuário
- acessar o conteúdo das notas;
- buscar alguma nota;
- alterar a ordem de alguma nota no vetor;
- alterar o texto de uma nota;
- alterar alguma informação adicional de uma nota;
- remover uma nota;
- criar uma nova nota.

Informações adicionais de uma nota:
- cor;
- retângulo com tamanho e posição na tela;
- etiqueta - 3 letras que servem para categorizar a nota.

A cor pode ser um registro com os valores dos componentes rgb entre 0 e 255.
O retângulo deve ser um registro, contendo um ponto (canto inferior esquerdo) e um tamanho (largura e altura).

Cada aluno pode escolher se suportará todas as informações adicionais ou um subconjunto delas, que pode ser:
- somente etiquetas;
- somente cor + retângulo.
A primeira opção é restrita aos alunos com deficiência visual que dificulte muito a manipulação/visualização dos retângulos.

O formato do arquivo de notas deve ser se acordo com o que segue. Caso o aluno opte por não suportar alguma das informações adicionais, ainda assim deve seguir o mesmo formado do arquivo, e essas informações devem ser mantidas no vetor e gravadas no arquivo. O arquivo contém uma nota por linha, com os seguintes campos, separados por um ou mais espaços:
- 3 caracteres com a etiqueta;
- cor rgb: 3 números entre 0 e 255, codificados em decimal, separados por um ou mais espaços;
- retângulo: 4 numeros inteiros codificados em decimal, representando as cordenadas x e y e a largura e altura;
- um número qualquer de caracteres, todos diferentes de '|', que devem ser ignorados;
- o caractere '|'
- os demais caracteres até o final da linha compõem o texto da nota.

Se o texto de alguma nota contiver mais caracteres que o suportado pelo programa, os caracteres a mais devem ser ignorados, mas o programa deve funcionar corretamente. A linha deve ser copiada para o final de um arquivo de linhas com problemas.

Se uma linha não estiver corretamente formatada, deve ser copiada para o final do arquivo de linhas com problemas, e ignorada pelo programa (que deve continuar lendo as demais linhas).

O vetor de notas e demais informações úteis para seu gerenciamento devem ser mantidos em um registro, que deve ser declarado na `main` e passado por referência para as funções que necessitem.

O programa deve ter um estrutura modal, controlado pelo teclado.
Um programa modal tem vários modos de funcionamento possíveis (por exemplo, modo de edição e modo de navegação), e a cada momento está em um desses modos.
Em um modo, o usuário pode realizar um conjunto de ações, selecionáveis pelo teclado; entre elas sempre está a ação de trocar de modo.
Cada modo é implementado como um laço:
```c
   enquanto estiver no modo tal:
   - desenha a tela
   - lê o teclado
   - executa a ação correspondente
```

Deve ter uma função para cada modo do operação do programa, implementando um laço como esse.
Deve ter uma função de desenho de tela para cada modo, que é chamada no laço correspondente.
Todo comando de saída para a tela deve estar nas funções de desenho de tela.
Deve ter uma função para cada ação possível.

O programa tem um laço principal assim:
```
  enquanto o modo não for "terminar"
  - chama a função correspondente ao modo atual
```

#### Modo principal

Esse modo permite a navegação entre as notas.

#### Modo de edição

Esse modo permite a alteração de uma nota.
